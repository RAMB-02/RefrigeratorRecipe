About Me

저는 좋은 개발 아이디어가 떠오르면 그걸 만들어 보는 실행력과 열정을 가지고 있습니다. '냉장고 해결사'는 제가 실생활에서 느꼈던 불편함을 해결하기 위해 Ollama와 Streamlit, 그리고 LangChain(RAG) 및 Chroma 벡터 스토어를 핵심으로 하여 개발되었습니다.

프로젝트 상세

냉장고 앞에서 뭘 먹을 지 고민하며 낭비한 시간을 냉장고 해결사가 대신 해결해줍니다. 냉장고 속 재료와 사용자의 선호 요리, 알레르기를 고려해, 만들 수 있는 요리의 이름, 재료, 과정을 알려줍니다.

아키텍처/설계

로컬에 저장된 레시피(`recipes.csv`)를 `Chroma` 벡터 DB로 구축하여 AI의 지식 기반으로 삼습니다. 사용자가 입력을 요청하면, LangChain RAG 파이프라인이 이 지식 기반을 참조하고 `Ollama(Mistral)` 모델이 최종 레시피를 생성합니다. 모든 사용자 정보(냉장고 재료, 저장된 레시피)는 사용자의 로컬(`user_profile.json`, `my_saved_recipes.csv`)에 저장하는 방식을 채택했습니다.

핵심 기능

아래 탭을 클릭하여 각 기능의 구현 방식을 확인하세요.

내 냉장고 (My Fridge)

`st.form`을 사용해 사용자의 냉장고 재료와 선호/알레르기 정보를 입력받습니다. '냉장고 정보 저장' 버튼을 누르면 `save_user_profile` 함수가 호출되어, 입력된 내용을 `user_profile.json` 파일에 덮어씁니다.

이 정보는 앱 세션(`st.session_state`)에 저장되며, '레시피 생성' 탭을 클릭할 때마다 기본값으로 자동 로드되어 사용자 편의성을 높였습니다.

레시피 생성 (Recipe Generation)

LangChain의 RAG(Retrieval-Augmented Generation) 파이프라인을 통해 구현되었습니다. 사용자가 '레시피 생성'을 요청하면, `setup_rag` 함수가 `Chroma` 벡터 스토어에서 `OllamaEmbeddings`를 사용해 관련 레시피(Context)를 검색(retrieve)합니다.

이 정보는 사용자의 냉장고 재료(`question`)와 함께 `ChatPromptTemplate`으로 전달됩니다. 최종적으로 `Ollama(Mistral)` 모델이 이 모든 정보를 바탕으로 "냉장고 재료만 100% 사용"과 같은 핵심 규칙을 준수하는 레시피 텍스트를 생성(generate)합니다.

나의 레시피 (My Recipes)

'저장된 레시피 보기'와 '지식 기반 추가' 두 탭으로 구성됩니다.

  • 저장된 레시피 보기: AI가 생성한 레시피 묶음을 저장하면 `my_saved_recipes.csv` 파일에 저장됩니다. `pandas`를 사용해 이 파일을 읽어 `st.dataframe`으로 목록을 표시합니다. 사용자가 `st.selectbox`로 특정 레시피 묶음을 선택하면 `st.markdown`을 통해 저장된 전체 텍스트를 확인할 수 있습니다. '삭제' 버튼 클릭 시 `delete_recipe` 함수가 `pandas`로 해당 행을 제외하고 CSV 파일을 다시 저장한 뒤 `st.rerun()`을 호출해 목록을 실시간 갱신합니다.
  • 지식 기반 추가 (RAG): 사용자가 `st.form`을 통해 새 레시피(이름, 재료, 과정)를 직접 입력하면, `append_to_rag_kb` 함수가 이 데이터를 `recipes.csv` (AI의 지식 기반) 파일에 `pandas`로 추가합니다. (이후 `rag_setup.py` 재실행 필요)
