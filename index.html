Project Detail

개인화된 레시피를 제안하는 AI 서비스

기획 의도 및 목표 기능

재료 낭비 방지

냉장고에 남은 자투리 재료를 100% 활용하여 음식물 쓰레기를 줄이는 것

초개인화 추천

사용자의 알레르기 정보와 싫어하는 재료를 완벽하게 필터링한 레시피 제공

데이터 주권 확보

Local LLM을 활용하여 모든 민감 정보가 외부로 유출되지 않는 환경 구축

아키텍처 & 설계

로컬 레시피 데이터(`recipes.csv`)를 Chroma 벡터 DB로 구축하여 AI의 지식 기반으로 활용합니다. 사용자 요청 시 LangChain RAG 파이프라인이 작동하며, Ollama(Mistral) 모델이 최종 레시피를 생성합니다. 데이터 프라이버시를 위해 모든 사용자 정보는 로컬 JSON/CSV 파일로 관리됩니다.

시스템 아키텍처 다이어그램

개발 기능 (핵심 구현)

탭을 클릭하여 상세 구현 방식을 확인하세요.

1

내 냉장고 (My Fridge)

st.form을 사용하여 사용자의 재료와 알레르기 정보를 수집합니다. 정보 저장 시 save_user_profile 함수가 JSON 파일에 데이터를 영속화합니다.

앱 세션(st.session_state)과 연동되어 재방문 시에도 이전 입력값이 자동으로 로드되어 끊김 없는 경험을 제공합니다.

2

레시피 생성 (Recipe Generation)

RAG (Retrieval-Augmented Generation) 핵심 엔진입니다. 사용자가 생성을 요청하면:

  1. setup_rag 함수가 Chroma 벡터 DB에서 관련 레시피를 검색(Retrieve)합니다.
  2. 검색된 Context와 사용자 냉장고 재료를 ChatPromptTemplate에 주입합니다.
  3. Ollama(Mistral) 모델이 "냉장고 재료 100% 사용" 규칙을 준수하며 최종 텍스트를 생성합니다.
3

나의 레시피 (My Recipes)

저장된 레시피 보기

CSV 파일(`my_saved_recipes.csv`)을 Pandas로 읽어와 목록을 표시합니다. 선택 시 전체 내용을 Markdown으로 렌더링하며, 삭제 기능도 지원합니다.

지식 기반 추가

사용자가 직접 입력한 레시피를 AI의 학습 데이터(`recipes.csv`)에 추가하여, 이후 RAG 검색 결과에 반영되도록 확장성을 고려했습니다.
